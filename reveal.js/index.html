DaCe — Introductionslide 1
contexte

Le problème de performance

  • Les programmes HPC sont complexes à optimiser
  • Les compilateurs classiques ont une vision locale
  • La mémoire est souvent le vrai goulot d'étranglement
  • Chaque architecture (CPU/GPU/FPGA) demande des optimisations différentes

Approche traditionnelle

Écrire du code → Compiler → Profiler → Réécrire à la main pour chaque cible. Répéter indéfiniment.

Approche DaCe

Décrire le quoi, pas le comment. Laisser DaCe générer du code optimisé pour la cible.

Une représentation exploitable

Le SDFG rend explicites les flux de données, les accès mémoire et les dépendances pour que les optimisations deviennent manipulables.

DaCe — Motivationslide 4
représentation interne

Le SDFG

  • États : nœuds du graphe externe (flot de contrôle)
  • Dataflow : graphe interne à chaque état (accès mémoire)
  • Symboles : paramètres symboliques (taille des tableaux…)
  • Descripteurs : type, forme et layout des données
  • Transformations applicables de façon automatique ou manuelle
États du SDFG
init → loop → exit
chaque état contient
un graphe de dataflow

Sérialisable en .sdfg (JSON), visualisable dans VS Code ou le web editor.

DaCe — SDFG IRslide 5
utilisation

DaCe en Python

import dace
import numpy as np

# Déclarer un symbole
N = dace.symbol('N')

# Annoter la fonction
@dace.program
def my_kernel(
    A: dace.float64[N],
    B: dace.float64[N]
):
    B[:] = A * 2.0

# Compiler en .so
sdfg = my_kernel.to_sdfg()
sdfg.compile()

# Appel direct
A = np.ones(100)
B = np.zeros(100)
my_kernel(A=A, B=B)

Compilation

DaCe génère du C++, le compile avec le compilateur système, et produit une .so réutilisable.

DaCe — Python APIslide 6
intégration

Appeler un SDFG depuis C

/* Header auto-généré par DaCe */
#include "my_kernel.h"

int main() {
    int N = 100;
    double *A = malloc(N * sizeof(double));
    double *B = malloc(N * sizeof(double));

    /* 1. Initialiser (symboles uniquement) */
    my_kernel_t handle = __dace_init_my_kernel(N);

    /* 2. Exécuter (handle + tableaux + symboles) */
    __program_my_kernel(handle, A, B, N);

    /* 3. Libérer les ressources */
    __dace_exit_my_kernel(handle);

    free(A); free(B);
    return 0;
}
DaCe — Intégration C/C++slide 7
API générée

Les 3 fonctions exportées

__dace_init_<nom>(symboles…)

Alloue toutes les structures internes. Ne prend que les symboles (pas les tableaux). Retourne un handle opaque.

__program_<nom>(handle, args…, symboles…)

Exécute le SDFG. Les arguments sont ordonnés alphabétiquement. Le handle est toujours le premier argument.

__dace_exit_<nom>(handle)

Libère toutes les ressources. Retourne 0 si succès. À appeler une fois le SDFG terminé.

DaCe — API Cslide 8
optimisation

Transformations de SDFG

  • MapTiling — découpage en tuiles cache-friendly
  • MapFusion — fusion de boucles adjacentes
  • Vectorization — génération SIMD automatique
  • GPUTransformMap — portage sur CUDA/HIP
  • InLocalStorage — promotion en registres/shared mem
  • Redundant copy elim. — suppression de copies inutiles

Application en Python

sdfg.apply_transformations(
